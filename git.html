gitの使い方

ファイルのあげ方

やることコマンド
ユーザー設定をする。初めて使う時だけやる。git config --global user.name "ユーザ名"
git config --global user.email "メールアドレス"
gitの初期化をする。gitの管理下に置きたいファイルで実行する。.gitのファイルが出来ているかかくにんをする。git init
コミット待ちの状態にする。.のあとに何も書かないと全部のファイルをやるgit add.
コミットする。git commit -m "コメント"
gitのリポジトリに登録する。-uは最初だけ入れる。originのあとは枝の名前git push (-u) origin master

ほかのコマンドメモ

説明コマンド
susiという枝を作るgit branch susi
susiという枝に移動git checkout susi
gitの状態を見るgit status
コミットのログを見るgit log

github-pagesの作り方

「GitHub Pages」は、「GitHub」を使ってホームページを公開することができる機能です。 GitHubの アカウント を持っていない場合はGitHub公式サイトから作成しする。
ホームページのトップに「 ユーザー 名」「メールアドレス」「パスワード」を入力することで、簡単に アカウント を作成することが可能。
ユーザー 名がGitHub Pagesの URL 「ユーザー名.github.io」になる。メールアドレスの認証まで済むと、実際にリポジトリの作成から行うことができる。
リポジトリとは、関連するファイルやフォルダをまとめておく、GitHubにおける貯蔵庫のようなもの。
リポジトリ名(Repogitory Name)の欄には、「ユーザー名.github.io」の形で入力し、ログイン後、リポジトリを新規作成する。

github-pagesの更新の仕方

git commit -m "コメント"でコミットしてgit push (-u) origin masterでプッシュする。
